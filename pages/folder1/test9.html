# 异步

异步编程的核心：程序中现在运行的部分和将来运行的部分之间的关系

# 理解异步编程

# 异步的概念

Ajax的异步请求： 虽然有存在“同步Ajax请求”，但是并不可取哦，同步的Ajax请求会锁定浏览器UI（按钮，菜单，滚动条），并阻塞所有的用户交互。

再来看看Ajax的异步请求： 首先来看看不使用异步：

  var data = ajax("http://some.url.1");
  console.log(data);
  //data 通常是不会包含Ajax的请求结果

Ajax不是同步完成的，data无法直接读取它的结果

当然，事实上我们也不是用这种方法进行Ajax请求： 我们需要发送一个异步的Ajax请求，然后在将来才能的到返回的结果： 如何实现呢？ 最最简单的方法，使用一个“回调函数”

  ajax("http://some.url.1",function myCallbackFunction(data){
    console.log(data);
    //这样，我们可以得到一些数据
  })

# 事件循环

先给出一段伪代码来助于理解：

// eventLoop 是一个用做队列的数组
//先进，先出
var eventLoop = [];
var event;

// "永远执行"
while (true) {
  //一次tick
  if (eventLoop.length > 0) {
    //拿到队列中的下一个事件
    event = eventLoop.shift();
    //现在，执行下一个事件
    try {
      event();
    }
    catch (err) {
      reportError(err);
    }
  }
}

有一个用while循环实现的持续运行的循环，循环的每一轮称为一个tick'，对每个tick而言，如果在队列中有等待事件，那么就会从队列中拿出下一个事件去执行，这些事件就是回调函数

setTimeOut(...)并没有把回调函数挂载事件循环队列中，它只是作为一个定时器，当到时间后，环境会把回调函数放在事件循环中，在未来，某个时刻tick会拿出这个回调去执行

# 并行和异步

  • 异步是关于未来和将来的时间间隙
  • 并行是关于能够同时发生的事情

# 任务队列

它是挂在事件循环队列的每个tick之后的一个队列，在事件循环的每个tick中，可能会出现的一部动作不会导致一个完整的新事件添加到事件循环队列中，而会在当前的tick任务队列末尾添加一个任务

好像在说：这里还有一件事将来要做，但要确保其他任何事情发生之前就完成它

# 小结

在js中，一旦有事件需要运行，事件循环就会运行，直到队列清空。事件循环的每一轮称为一个tick。用户交互，IO，定时器会想事件队列中加入事件
